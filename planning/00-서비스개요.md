# Scanly 서비스 개요

## 핵심가치

### 1. 간편함 (Simplicity)
- 한 번의 스캔으로 명함 교환 완료
- 복잡한 설정 없이 바로 사용 가능
- 최소한의 필수 정보만 요구

### 2. 양방향 연결 (Mutual Connection)
- 일방적 명함 전달이 아닌 상호 교환
- "내 명함도 전송" 기능으로 자연스러운 네트워킹
- 받는 사람도 주는 사람도 가치를 얻는 구조

### 3. 실시간성 (Real-time)
- 명함 정보 변경 시 즉시 반영
- 재인쇄, 재배포 불필요
- 항상 최신 정보 유지

### 4. 가벼움 (Lightweight)
- 빠른 앱 실행과 스캔
- 불필요한 기능 배제
- 핵심 기능에 집중

---

## 서비스 원칙

### 사용자 경험
- 3초 안에 스캔 → 저장 완료 가능해야 함
- 회원가입 없이도 명함 열람 가능
- 오프라인 환경에서도 내 QR 코드 표시 가능

### 데이터
- 사용자 데이터는 사용자의 것
- 명함 정보는 언제든 수정/삭제 가능
- 개인정보는 최소한만 수집

### 접근성
- 비회원도 명함 열람 가능
- 앱 미설치자도 웹으로 명함 확인 가능

## 용어 정의

| 용어 | 정의 |
|------|------|
| 회원 (Member) | Scanly에 가입한 사용자 |
| 명함 (Card) | 회원이 생성한 자신의 디지털 명함 |
| 명함첩 (CardBook) | 저장한 타인의 명함 모음 |
| 저장된 명함 (SavedCard) | 명함첩에 저장된 개별 명함 |
| 스캔 (Scan) | QR 코드를 인식하여 명함을 조회하는 행위 |
| 상호 교환 | 명함 저장 시 내 명함도 상대방에게 전송하는 기능 |

## 개발 로드맵

### Phase 1: MVP
- 회원가입/로그인
- 명함 생성/수정
- QR 코드 생성/공유
- QR 스캔
- 명함 저장 + 상호 교환

### Phase 2: 핵심 기능
- 명함첩 (그룹/태그)
- 스캔 기록
- 푸시 알림
- 연락처 동기화

### Phase 3: 확장
- 스캔 통계
- 미팅 예약
- 캘린더 연동
